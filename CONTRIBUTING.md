# Contributing to Awesome GitHub README

Thanks for helping make this the best README resource on GitHub.

## Ways to Contribute

### 1. Add Your Profile

**Option A — Open an issue (easiest):**

[**Submit your profile here →**](https://github.com/yerdaulet-damir/awesome-github-readme/issues/new?template=add-profile.yml)

**Option B — Pull request:**

1. Fork this repository
2. Add your profile to the `## ⭐ Featured Profiles` section in `README.md` using this format:

```markdown
### @username

> *One-liner about you*

![Profile](https://coolreadme.xyz/api/cinematic?user=username&status=Your%20Status&focus=Your%20Focus&accent=A78BFA)

**Tools used:** `tool-1` · `tool-2` · `tool-3`

[**→ View Full Profile**](https://github.com/username)
```

3. Optionally, add a detailed profile page in `profiles/username.md`
4. Submit a pull request

**Once featured**, add this badge to your own profile README:

```markdown
[![Featured in Awesome GitHub README](https://img.shields.io/badge/Featured%20in-Awesome%20GitHub%20README-A78BFA?style=for-the-badge)](https://github.com/yerdaulet-damir/awesome-github-readme)
```

---

### 2. Add a Tool

**Option A — Open an issue:**

[**Suggest a tool here →**](https://github.com/yerdaulet-damir/awesome-github-readme/issues/new?template=add-tool.yml)

**Option B — Pull request:**

Add your tool to the appropriate section in `README.md`:

```markdown
| [**tool-name**](https://github.com/user/repo) | ![Stars](https://img.shields.io/github/stars/user/repo?style=social) | Brief description |
```

### Categories

- Profile Card Generators
- Stats & Metrics
- Animations & Effects
- Tech Stack & Skills
- Social & Activity
- Music, Media & Lifestyle
- Achievements & Trophies
- Badges & Icons

---

### 3. Add a Template

1. Create a new file in `templates/` following the existing format
2. Add it to the Templates table in `README.md`
3. Include: preview, copy-paste code block, tools used, "best for" section

---

## Guidelines

- Tools must be actively maintained (commits in the last 6 months)
- Include a working preview or example
- Keep descriptions concise — one line for tools, one-liner for profiles
- Link to the GitHub repo when available
- No duplicates — check if the tool already exists
